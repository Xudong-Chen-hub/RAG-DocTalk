# Core Work Page — RAG-DocTalk

## 页面定位

核心工作界面，三段式布局。这是用户上传文档、提问、获得答案、生成多模态内容的主战场。从笔记管理页点击任意笔记进入。

## 逐区域详细设计

### 1. Top Bar 顶部栏

| 属性 | 值 |
|------|-----|
| 高度 | `48px` |
| 背景 | 浅色 `#FFFFFF` / 深色 `#242724`，底部 `1px solid var(--border)` |
| 左侧 | 返回箭头图标(20x20, `#6B7268`) → hover `#6B9B5E`，点击回笔记管理页。旁边笔记标题（思源宋体 16px weight 600 `#1B1D1A`），可点击重命名 |
| 右侧 | "设置" 按钮 + "分享" 按钮，思源黑体 13px `#6B7268`，hover `#1B1D1A`。间距 20px |

### 2. 面板通用规则

- 三个面板之间用 `1px solid var(--border)` 分隔
- 拖拽分隔线可调整面板宽度（`cursor: col-resize`）
- 最小宽度：Source `180px`，Community `320px`，Studio `200px`
- 面板背景与页面基底同色，内部卡片使用 `--bg-surface`

### 3. Source 面板（左侧 ~25%）

来源管理区。用户在此添加和管理知识来源。

#### 3.1 顶部操作区

| 元素 | 样式 |
|------|------|
| "+ 添加来源" 按钮 | 全宽，`border: 1px dashed var(--border)`，`border-radius: 8px`，`padding: 12px`，文字思源黑体 14px `#6B7268`。hover 时 `border-color: #6B9B5E`，文字变 `#6B9B5E` |
| 点击后展开下拉菜单 | 三个选项：上传文档 / 粘贴文本 / 联网搜索。思源黑体 14px，图标+文字 |

#### 3.2 来源分组列表

| 分组 | 展开/折叠 |
|------|-----------|
| ▼ 文档 | 展开（默认），列出所有上传的文档文件 |
| ▼ 文本 | 展开（默认），列出所有手动输入的文本 |
| ▼ 搜索 | 展开（默认），列出所有联网搜索导入的来源 |

每个来源项的样式：

| 属性 | 值 |
|------|-----|
| 高度 | `40px` |
| padding | `8px 12px` |
| 左侧 | 文件类型图标（16x16，`#8B9288`）：PDF/Word/TXT/文本/网页，各不同图标 |
| 中间 | 文件名（思源黑体 13px `#1B1D1A`，单行截断） |
| 右侧 | 删除图标（14x14，`#8B9288`），hover 变 `#C2544A` |
| hover | 背景 `var(--bg-hover)`，`border-radius: 6px` |

#### 3.3 上传交互

| 步骤 | 表现 |
|------|------|
| 拖拽文件到 Source 面板 | 面板边框变为 `#6B9B5E`，显示虚线框 + "释放以上传" |
| 上传中 | 来源列表顶部出现进度条：文件名 + 细线进度条（`height: 2px`，背景 `#E2E5DF`，填充 `#6B9B5E`）+ "解析中..." |
| 完成 | 进度条消失，来源项出现在对应分组，带微妙的入场过渡（`opacity 0→1`，200ms ease-out） |
| 失败 | 进度条变红 `#C2544A`，显示 "解析失败：文件可能已损坏" |

#### 3.4 文本输入弹窗

| 属性 | 值 |
|------|-----|
| 触发 | 点击 "粘贴文本" |
| 外观 | Modal，`max-width: 480px` |
| 内容 | 标题 "添加文本来源" + textarea（`min-height: 160px`，placeholder "粘贴或输入文本内容..."）+ 来源标题(可选) + "确认添加" 按钮 |

#### 3.5 联网搜索

| 属性 | 值 |
|------|-----|
| 触发 | 点击 "联网搜索" |
| 外观 | Modal，`max-width: 560px` |
| 内容 | 搜索框 + 搜索结果列表(每条结果含标题+摘要+勾选框) + "导入选中" 按钮 |
| 搜索结果项 | `border: 1px solid var(--border)`，`border-radius: 8px`，`padding: 12px`。hover 边框变 `#6B9B5E` |

### 4. Community 面板（中间 ~45%）

对话交互区，产品的核心体验。

#### 4.1 空状态（新笔记，无对话）

- 居中：笔记本图标（SVG 描边风格，64x64，鼠尾草绿灰度）+ "向你的文档提问"（思源宋体 22px weight 600 `#1B1D1A`）
- 副行："你的问题会基于已添加的来源进行回答。"（思源黑体 15px `#6B7268`）
- 下方三个建议问题按钮(仅当有来源时显示)，思源黑体 14px，`border: 1px solid var(--border)`，`border-radius: 20px`（药丸形），`padding: 8px 16px`。hover `border-color: #6B9B5E`。示例："总结这份文档的主要内容" / "文档中提到了哪些关键数据" / "这份合同有哪些需要注意的条款"

#### 4.2 消息列表

| 属性 | 值 |
|------|-----|
| 滚动 | `overflow-y: auto`，新消息自动滚动到底部（`scroll-behavior: smooth`） |
| 内边距 | `padding: 24px` |
| 消息间距 | 消息之间 `margin-bottom: 24px` |

#### 4.3 用户消息气泡

| 属性 | 值 |
|------|-----|
| 对齐 | 右对齐，`max-width: 75%` |
| 背景 | `var(--bubble-user)` |
| 内边距 | `12px 16px` |
| 圆角 | `12px`（右上角 `4px`，形成方向性暗示） |
| 字体 | 思源黑体 15px，`#1B1D1A` |
| 时间戳 | 气泡右侧 8px，12px `#8B9288` |

#### 4.4 AI 消息气泡

| 属性 | 值 |
|------|-----|
| 对齐 | 左对齐，`max-width: 85%` |
| 背景 | `var(--bubble-ai)` |
| 内边距 | `16px 20px` |
| 圆角 | `12px`（左上角 `4px`） |
| 字体 | 思源黑体 15px，`#1B1D1A`，`line-height: 1.7` |
| 内容渲染 | Markdown：段落、列表、粗体、斜体、代码块（JetBrains Mono 13px，`background: var(--bg-hover)`） |

#### 4.5 引用标记（核心交互）

这是产品质量的关键差异点。

| 属性 | 值 |
|------|------|
| 内联标记 | AI 回答中的引用编号：`[1]` `[2]` `[3]`。鼠尾草绿色文字 `#6B9B5E`，思源黑体 12px weight 500，`vertical-align: super` |
| 点击/hover 引用 | 右侧弹出引用卡片（非 tooltip，是固定位置的侧边卡片） |
| 引用卡片 | 宽度 `320px`，背景 `#FFFFFF`，`border: 1px solid var(--border)`，`border-radius: 10px`，`padding: 16px`，`box-shadow: 0 4px 12px rgba(27,29,26,0.06)` |
| 卡片内容 | 顶部文件名(思源黑体 12px weight 600 `#6B7268`) + 原文片段（思源宋体 14px `#1B1D1A`，`line-height: 1.8`）+ 高亮对应句（`background: var(--cite-highlight)`，`padding: 2px 0`） |
| 关闭 | 点击卡片右上角 ×，或点击面板空白处 |

#### 4.6 输入区域

| 属性 | 值 |
|------|-----|
| 位置 | Community 面板底部，固定 `position: sticky; bottom: 0` |
| 背景 | `var(--bg-surface)`，顶部 `1px solid var(--border)` |
| 内边距 | `16px 20px` |
| 输入框 | 多行 textarea，`min-height: 44px`，`max-height: 160px`（自动扩高），`border: 1px solid var(--border)`，`border-radius: 10px`，`padding: 10px 44px 10px 16px`，思源黑体 15px。focus `border-color: #6B9B5E` |
| placeholder | "向你的文档提问..."（不在输入框中显示字数统计） |
| 发送按钮 | 绝对定位在输入框右下角，圆形(32x32)，背景 `#6B9B5E`，白色箭头图标。hover `#597F4E`。输入为空时 `opacity: 0.3`，有内容时 `opacity: 1` |
| 停止生成按钮 | 流式生成中，发送按钮变为 ■ 停止按钮（背景 `#C2544A`） |

#### 4.7 流式生成效果

- AI 回答通过 SSE 逐字呈现
- 新消息气泡从空白开始，文字逐字符填入（不是 typewriter 动画，是直接的 text append）
- 引用标记 `[N]` 在对应句子生成完毕后出现
- 生成完成时，引用标记变为可点击态（`cursor: pointer`）

### 5. Studio 面板（右侧 ~30%）

多模态输出工具。仅在用户主动触发时执行。

#### 5.1 生成工具区

三个工具按钮，竖向排列：

| 按钮 | 图标 | 标题 | 描述 |
|------|------|------|------|
| 🎵 音频概述 | 声波 SVG | 生成音频概述 | 将文档内容转为双人播客对话 |
| 🧠 思维导图 | 节点 SVG | 生成思维导图 | 基于文档结构生成可交互脑图 |
| 📊 PPT 导出 | 幻灯片 SVG | 导出 PPT | 生成演示文稿文件 |

每个按钮样式：

| 属性 | 值 |
|------|-----|
| 宽度 | 全宽 |
| padding | `14px 16px` |
| 边框 | `1px solid var(--border)` |
| 圆角 | `10px` |
| 布局 | flex，图标在左（24x24），标题+描述在右 |
| 标题 | 思源黑体 15px weight 500 `#1B1D1A` |
| 描述 | 思源黑体 12px `#8B9288`，一行 |
| hover | `border-color: #6B9B5E`，背景 `var(--bg-hover)` |
| 间距 | 按钮之间 `12px` |

**交互**：点击后按钮变为 "生成中..." + 旋转加载图标(18x18, `#6B9B5E`)，不可再次点击。完成后状态更新为 "已完成" + ✓ 图标。

#### 5.2 生成文件列表

工具区下方 24px 处开始。标题 "生成文件"（思源宋体 15px weight 600 `#1B1D1A`）。

每个文件项：

| 属性 | 值 |
|------|-----|
| 高度 | `44px` |
| padding | `8px 12px` |
| 左侧 | 文件类型图标(16x16)：🎵(音频) / 🧠(脑图) / 📊(PPT) |
| 中间 | 文件名(思源黑体 13px) + 生成时间(12px `#8B9288`) |
| 右侧 | 下载按钮(16x16, `#6B7268`，hover `#6B9B5E`) + 删除按钮(16x16, `#8B9288`，hover `#C2544A`) |
| 空状态 | "暂无生成文件。选择一个工具开始。" (思源黑体 13px `#8B9288`，居中) |

### 6. Status Bar 底部状态栏

| 属性 | 值 |
|------|-----|

## 与 NotebookLM 的差异总结

| NotebookLM | RAG-DocTalk |
|-------------|-------------|
| Google Material Design 蓝灰 | 鼠尾草绿 + 石板灰书房调 |
| 仅 sans-serif（Google Sans / Roboto） | 思源宋体标题 + 思源黑体正文 |
| "音频概览" 是招牌功能 | 音频概览是工具之一，平等对待 |
| 分享/协作功能突出 | 个人使用为主，分享位于 Top Bar 次要位置 |
| 九宫格应用入口 | 无（独立产品） |
| 蓝底 CTA 按钮 | 鼠尾草绿 CTA 按钮 |

## 响应式

| 断点 | 变化 |
|------|------|
| ≥1200px | 三面板完整显示 |
| 960–1200px | Studio 面板收起为图标栏(56px)，点击展开 |
| 640–960px | Source 面板收起，通过左上角汉堡菜单呼出；Community 全宽 |

## 动效

| 元素 | 动效 |
|------|------|
| 面板宽度拖拽 | 实时跟随光标，无动画 |
| 消息出现 | `opacity: 0→1` + `translateY(4px→0)`，200ms ease-out |
| 引用卡片 | `opacity: 0→1`，150ms ease-out。不使用 scale 变换 |
| 来源添加 | 列表项 `opacity: 0→1`，200ms ease-out |
| Studio 生成状态 | 图标旋转（`@keyframes spin`，`animation: spin 1s linear infinite`） |
| 面板切换（移动端） | 无过渡动画，即时切换 |
| `prefers-reduced-motion` | 所有过渡禁用 |

## AI Slop 自检

- ❌ 无 glassmorphism 面板背景
- ❌ 无 gradient text（即使是引用标记 [N] 也用纯色）
- ❌ 无嵌套卡片（消息气泡不是卡片，是独立区域）
- ❌ 无 side-stripe border 标记选中态
- ❌ 无 cream/sand 底色
- ❌ 无 em-dash、无 buzzword
- ❌ 无全大写 eyebrow
