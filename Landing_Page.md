## 逐区域详细设计

### 1. Nav 导航栏

- 高度 `56px`
- 初始背景透明，滚动超过 64px 后切换为 `rgba(250,251,248,0.85)` + `backdrop-filter: blur(12px)`，过渡 300ms ease-out
- 左侧：Logo 图标 (鼠尾草绿 SVG) + "RAG-DocTalk" (思源宋体 20px weight 700 `#1B1D1A`)
- 右侧：两个链接 "关于" "文档" (思源黑体 14px `#6B7268`，hover → `#1B1D1A`)，间距 32px
- Nav 与 Hero 之间无可见分割线

### 2. Hero 区域

**核心设计决策：无大图、无渐变、无动画标题。**

| 属性 | 值 |
|------|-----|
| 内边距 | `120px 0 80px` |
| 背景 | `#FAFBF8` |
| 标题 | "让文档开口说话"，思源宋体，`clamp(2.5rem, 5vw, 4.5rem)`，weight 700，`#1B1D1A`，`letter-spacing: -0.02em`，`text-wrap: balance` |
| 装饰线 | 标题下方 24px 处，一条鼠尾草绿横线。`width: 60px; height: 3px; background: #6B9B5E; border-radius: 2px; margin: 0 auto`。不需要图片，纯 CSS |
| 副标题 | "上传你的文档，用自然语言提问，获得有据可查的回答。"，思源黑体 18px weight 400 `#6B7268`，`max-width: 480px`，`text-wrap: pretty` |
| 装饰线-副标题间距 | 24px |
| CTA 按钮文案 | "开始阅读"（不是"立即免费试用"或"开始使用"） |
| 按钮样式 | 背景 `#6B9B5E`，hover `#597F4E`，白色文字，思源黑体 16px weight 500，`padding: 12px 32px`，`border-radius: 8px`。无阴影，无渐变。hover 过渡 200ms ease-out |
| 按钮位置 | 副标题下方 40px，水平居中 |
| 文字对齐 | 全部居中 |
| 最大宽度 | 整体 `max-width: 640px`，水平居中 |

**Hero 设计核心**：不依赖任何图片。超大标题 + 一条鼠尾草绿的装饰线 + 副标题 + CTA 按钮 + 大量留白。NotebookLM 就是这样做的——排版即视觉。

**CTA 文案选择理由**："开始阅读" 是动词+对象的祈使句，暗示"阅读文档"的产品内核，与书房调性一致。"开始使用"太泛，"立即免费试用"是 SaaS 套话。

### 3. 功能展示区域

**不使用等大三列卡片网格。采用左右交错的图文布局，奇数行文字在左、图形在右，偶数行反向。**

**图形来源分两阶段**：
- **Phase 2**：用 CSS 绘制的抽象几何图形作为"界面骨架"占位
- **Phase 4+**：替换为真实产品界面截图（倾斜 `-2deg`，加极淡阴影）

每个功能区块：

| 属性 | 值 |
|------|-----|
| 内边距 | `80px 0` |
| 最大宽度 | `960px`，水平居中 |
| 标题 | 思源宋体 28px weight 600 `#1B1D1A` |
| 描述 | 思源黑体 15px `#6B7268`，`max-width: 380px`，`line-height: 1.7` |
| 布局 | `flex`，`gap: 80px`。偶数行 `flex-direction: row-reverse` |

三个功能区块的文案：

| # | 标题 | 描述 |
|---|------|------|
| 1 | 把文档变成对话 | 上传 PDF、Word 或纯文本。系统自动分块、理解内容，然后你就可以像和人聊天一样，向你的文档提问。 |
| 2 | 每个回答都有据可查 | 不是凭空生成。每一条回答都标注了原文出处，点击引用即可看到原始段落。你知道答案从哪里来。 |
| 3 | 不止于问答 | 把文档内容转为双人播客音频、思维导图、PPT。你需要的时候，一键生成。不需要的时候，它安静地等着。 |

**CSS 几何占位图设计（Phase 2 使用）**：

三个图形均为纯 CSS 渲染，不用任何图片文件。风格统一：圆角矩形轮廓 + 品牌色点缀 + 模拟界面骨架。

| 图形 | CSS 实现要点 |
|------|-------------|
| 占位 1（文档→对话） | 左侧：一个 `#E2E5DF` 边框的圆角矩形 + 内部 4 条 `#6B9B5E` 短线代表文字行。右侧：一个对话气泡轮廓（`border-radius: 12px` 的圆角矩形 + CSS 三角 `::after`）。两者之间用一条 `#6B9B5E` 虚线连接（`border-left: 1.5px dashed`） |
| 占位 2（引用溯源） | 左侧：圆角矩形 + 内部一段 `rgba(107,155,94,0.15)` 高亮色条。从高亮条右侧延伸一条 `#6B9B5E` 弧线（`border` + `border-radius` 画半圆弧）到右侧的小圆角引用标记（`border-radius: 4px` 小矩形 + 内容 "[1]"） |
| 占位 3（多模态放射） | 中心：一个 `#1B1D1A` 填充的深色圆角矩形。用 CSS Grid 在四角放置四个白色小面板（`border: 1px solid #E2E5DF`），每个面板内部一个 Lucide 图标。中心到各面板用 `::before` 伪元素画连接线 |

**产品截图替换（Phase 4+）**：

截图样式：`border-radius: 10px; box-shadow: 0 1px 3px rgba(27,29,26,0.06); transform: rotate(-2deg); background: #F6F7F4`。

### 4. Footer

- `padding: 48px 0`，背景 `#FFFFFF`（与内容区形成微妙层次）
- 左侧 "RAG-DocTalk © 2026"，右侧 "GitHub" + "反馈" 链接
- 思源黑体 13px `#8B9288`，`display: flex; justify-content: space-between; max-width: 960px; margin: 0 auto`

## 响应式行为

| 断点 | 变化 |
|------|------|
| ≥960px | 完整布局 |
| 640–960px | Hero 字体缩小；功能展示改为上下排列（不交错），图+文各占一行 |
| <640px | Nav 仅显示 Logo；Hero `padding: 80px 0 60px`；功能展示 `padding: 48px 0`；CTA 按钮宽度 100% |

## 动效

| 元素 | 动效 |
|------|------|
| Nav 背景 | 滚动触发透明→毛玻璃，300ms ease-out |
| CTA 按钮 | hover 背景过渡，200ms ease-out |
| 功能展示 | **无入场动画**，页面加载即完整展示 |
| 装饰线 | 页面加载时从左到右展开（`scaleX 0→1`，600ms ease-out-quart，仅一次）。如果用户偏好减少动效，则直接显示 |
| `prefers-reduced-motion` | 所有过渡禁用，瞬间切换 |

## AI Slop 自检清单

- ❌ 无 Hero 大图、渐变背景、打字机动效
- ❌ 无 AI 生成的装饰插画（零外部图片依赖）
- ❌ 无 "ABOUT" "FEATURES" 等全大写 eyebrow
- ❌ 无 01 / 02 / 03 编号 section 标记
- ❌ 无 等大三列 icon+heading+描述 卡片网格
- ❌ 无 统计数据大数字
- ❌ 无 glassmorphism
- ❌ 无 cream/sand/beige 底色
- ❌ 无 em-dash、aphoristic 句式
- ❌ 无 "赋能" "解锁" "超级" "无缝" 等 buzzword

## 本次更新的关键变更

- **去掉所有 AI 插画** → 走纯排版路线：Hero 靠文字+装饰线，功能展示区先 CSS 几何占位再替换真实截图
- **唯一外部生成** → Logo 图形和 favicon，这两个简单到 AI 不会搞砸
- **功能图标** → 全部用 Lucide 开源图标库，不自己画
